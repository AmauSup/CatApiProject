# 001 – Choix Node.js + Express

## Contexte
Le projet nécessite un backend léger, rapide à mettre en place, adapté à la création d’API REST et bien documenté. L’équipe a des compétences en JavaScript.

## Décision
Utiliser Node.js (JavaScript) avec le framework Express pour développer l’API backend.

## Conséquences
- Rapidité de prototypage et d’itération
- Large écosystème de packages (npm)
- Facilité d’intégration avec des bases SQL et des middlewares

## Alternatives considérées
- Python (Flask, Django)
- Java (Spring Boot)
