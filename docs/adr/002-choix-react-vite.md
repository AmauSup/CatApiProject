# 002 – Choix React + Vite

## Contexte
Le frontend doit être réactif, moderne, et permettre une bonne séparation des composants. L’équipe souhaite utiliser une technologie populaire et bien supportée.

## Décision
Utiliser React pour le frontend, avec Vite comme outil de build et de développement rapide.

## Conséquences
- Hot reload rapide
- Large communauté et documentation
- Facilité de création de SPA

## Alternatives considérées
- Vue.js
- Angular
