# 005 – Choix API REST

## Contexte
L’application nécessite une interface standardisée pour la communication entre frontend et backend, facilement testable et documentable.

## Décision
Adopter l’architecture REST pour l’API, avec des routes claires et des verbes HTTP adaptés.

## Conséquences
- Facilité d’intégration avec des outils externes (Swagger, Postman)
- Documentation automatique possible
- Séparation claire des responsabilités

## Alternatives considérées
- GraphQL
- gRPC
